## Key Design Decisions

### 1. Sucrase over Babel
- **Why**: Faster transformation, smaller bundle
- **Trade-off**: Less plugin support, but sufficient for JSX/TS

### 2. Function Constructor for Eval
- **Why**: Safer than `eval()`, allows scope injection
- **Security**: Code runs in browser sandbox, no server execution

### 3. localStorage for Persistence
- **Why**: Simple, no backend required
- **Limitation**: ~5MB storage limit, single device

### 4. Monaco over CodeMirror
- **Why**: Better TypeScript support, VS Code familiarity
- **Trade-off**: Larger bundle size (~2MB)

### 5. ErrorBoundary Pattern
- **Why**: Prevents entire app crash from component errors
- **Behavior**: Catches errors, displays friendly message, auto-resets

### 6. Console Proxy Pattern
- **Why**: Capture console output without overriding global
- **Implementation**: Create proxy object, pass to preview scope
